# Courier Shift Bot

Telegram-бот для учёта смен курьеров, пробега, маршрутных листов и сверок.

Вся работа идёт внутри обычного Telegram-чата с ботом: кнопки, текст и фото. Mini App/Web App не используются.

## Что умеет бот

- Авторизует сотрудника по ФИО из Google Таблицы.
- Хранит профиль сотрудника: ФИО, номер машины, интернет-магазин, устройство.
- Записывает начало/конец смены за текущий день.
- Записывает пробег (старт/конец), валидирует формат и защищает от «конец меньше старта».
- Прикрепляет фото пробега в рабочий чат (или тему).
- Принимает фото маршрутных листов (можно несколько подряд).
- Принимает фото сверок (для `Терминал` — 2 фото, для `Пин-Панель` — 1 фото).
- Показывает статус сотрудника за сегодня: время и пробег.
- Поддерживает помесячные таблицы по магазинам (`ИМ Восток` и `ИМ Центр`) через `/sheet`.
- Рассылает пользователям сообщение об обновлении версии после изменения кода.

## Памятка сотруднику (короткая)

1. Нажмите `/start`.
2. Введите ФИО как в таблице.
3. Укажите номер машины, магазин и устройство.
4. Для начала/конца смены нажимайте `⏱ Время смены`.
5. Для пробега нажимайте `🚗 Пробег` и отправляйте фото одометра.
6. Если распознавание не удалось, используйте `📷 Загрузить фото повторно` или `✏️ Ввести вручную`.
7. Для проверки записей нажмите `📊 Статус`.

## Памятка сотруднику (подробно)

### 1) Первый вход

- Команда `/start` запускает авторизацию.
- Бот попросит ФИО, затем номер машины, магазин и устройство.
- Пока профиль не заполнен, рабочие кнопки (`Время`, `Пробег`, `Сверки` и т.д.) не выполняются.

### 2) Время смены

- Нажмите `⏱ Время смены`.
- Если за сегодня не заполнен старт — бот запишет старт.
- Если старт есть, а конец пустой — бот запишет конец.
- Если уже заполнены оба значения — бот предложит, что заменить: старт или конец.
- После автозаписи можно нажать `✏️ Изменить время` и ввести время вручную.

Допустимый ручной формат времени (минуты автоматически округляются до ближайших 30):

- `9` — целое число часов
- `9,5` или `9.5` — половина часа
- `09:30`, `08:46`, `8 14`, `08.46` — часы и минуты в любом формате
- Любой ввод сводится к виду `H` или `H,5` (например, `08:46` → `9`, `7,3` → `7,5`)

### 3) Пробег

- Нажмите `🚗 Пробег`.
- Бот определит этап (начало или конец смены) и попросит фото одометра.
- Фото отправляется в рабочий чат (если `WORK_CHAT_ID` настроен).
- Дальше бот пытается распознать пробег автоматически.
- Если не распознал: можно отправить фото ещё раз, ввести вручную или пропустить.

Ограничения:

- Пробег только от `2` до `6` цифр.
- Для конца смены значение не может быть меньше старта.

### 4) Маршрутный лист

- Нажмите `📄 Маршрутный лист` и отправьте фото.
- Можно отправлять несколько фото подряд.
- Бот присваивает номер маршрутного листа по сотруднику и текущей дате.

### 5) Сверки

- Нажмите `📊 Сверки`.
- Если устройство `Терминал`, бот ждёт 2 фото: `Терминал`, затем `Пин-Панель`.
- Если устройство `Пин-Панель`, бот ждёт 1 фото.

### 6) Настройки

В разделе `⚙️ Настройки` можно:

- изменить номер машины;
- изменить интернет-магазин;
- изменить устройство;
- сменить сотрудника (старый профиль будет удалён).

## Команды сотрудника

- `/start` — запуск/повторный вход.
- `/help` — краткая инструкция.
- `/status` — статус за сегодня.
- `/car` — изменить номер машины.
- `/workplace` — изменить магазин.
- `/device` — изменить устройство.
- `/cancel` — вернуться в меню и сбросить текущий шаг.
- `/chatid` — показать `chat_id` и `message_thread_id` текущего чата/темы.

## Команды администратора

Команда `/sheet` доступна только пользователям из `ADMIN_IDS`.

Если `ADMIN_IDS` не задан, команда отключена.

### Просмотр привязок

- `/sheet`
- `/sheet list`

Показывает:

- активный и следующий месяц;
- привязанные таблицы по каждому магазину;
- источник таблицы (`monthly`, `legacy`, `fallback`);
- предупреждения, если на активный месяц таблица не задана.

### Привязка таблиц

- `/sheet east active URL`
- `/sheet east next URL`
- `/sheet center active URL`
- `/sheet center next URL`
- `/sheet east 2026-06 URL`
- `/sheet center URL` (автоопределение месяца по названию таблицы)

`URL` можно передать как полную ссылку Google Sheets или сразу как `sheetId`.

### Сброс привязок

- `/sheet reset east active`
- `/sheet reset east next`
- `/sheet reset east 2026-06`
- `/sheet reset east all`
- `/sheet reset all`

## Таблицы и логика записи

Поддерживаются два магазина:

- `ИМ Восток`
- `ИМ Центр`

Требуемые листы:

- для времени: `Курьеры`
- для пробега:
  - `ИМ Восток` -> `Пробег`
  - `ИМ Центр` -> `Пробег Курьеры`

### Какие ячейки меняет бот

- Бот не добавляет строки (`append` не используется).
- Бот обновляет только конкретные ячейки через `spreadsheets.values.update`.
- Время: только старт/конец за текущий день в листе `Курьеры`.
- Пробег: только старт/конец за текущий день в листе пробега.
- Формулы, итоги, другие листы и прочие столбцы не трогаются.

### Округление времени

- `00-14` минут -> текущий час
- `15-44` минут -> `час,5`
- `45-59` минут -> следующий час

Примеры:

- `12:14` -> `12`
- `12:30` -> `12,5`
- `12:45` -> `13`

### Как выбирается таблица

Порядок разрешения таблицы для магазина:

1. Помесячная привязка (`/sheet ... active/next/YYYY-MM`).
2. Legacy-привязка (старый формат в `users.json`, если есть).
3. `GOOGLE_SHEET_ID` как fallback.

Важно: если для магазина уже создана помесячная карта, но нет таблицы на текущий месяц, запись блокируется до привязки нужного месяца.

## Распознавание пробега

Пайплайн распознавания:

1. RapidOCR (если включён).
2. Tesseract OCR (если включён).
3. Локальная проверка качества и согласования результатов.
4. Если локально неуверенно — fallback в Gemini Vision.
5. Если не распознано — ручной ввод сотрудником.

Если Gemini не настроен или модель временно недоступна, бот предлагает ввести пробег вручную.

Защита от ложных срабатываний локального OCR (по умолчанию):

- `OCR_MIN_MILEAGE=5000`
- `OCR_MAX_SHIFT_DELTA=800`

## Настройка Telegram-бота

1. Откройте `@BotFather`.
2. Создайте бота (`/newbot`).
3. Сохраните токен и запишите его в `.env` как `BOT_TOKEN`.

## Настройка Google Sheets API

1. В Google Cloud Console включите `Google Sheets API`.
2. Создайте Service Account.
3. Скачайте JSON-ключ.
4. Перенесите в `.env`:
   - `GOOGLE_SERVICE_ACCOUNT_EMAIL`
   - `GOOGLE_PRIVATE_KEY`

Пример `GOOGLE_PRIVATE_KEY` в одну строку:

```env
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

## Доступ к Google Таблицам

Для каждой таблицы, с которой работает бот:

1. Нажмите `Share`.
2. Добавьте `GOOGLE_SERVICE_ACCOUNT_EMAIL`.
3. Дайте роль редактора.

### Если без VPN бот не отвечает

Иногда провайдер блокирует или нестабильно маршрутизирует `api.telegram.org`. Тогда бот не может подключиться к Telegram и в логах появляется `ETIMEDOUT`.

Что делать:

1. Проверьте доступ:

```powershell
Test-NetConnection api.telegram.org -Port 443
```

2. Если недоступно без VPN — задайте прокси в `.env`:

```env
TELEGRAM_PROXY_URL=socks5://login:password@host:port
```

Поддерживаются `socks5://`, `http://`, `https://`.
После изменения перезапустите бота (`pm2 restart courier-shift-bot --update-env`).

Windows-скрипты:

- `start-bot.cmd` / `start-bot.ps1` — запуск в фоне с логами.
- `stop-bot.cmd` / `stop-bot.ps1` — корректная остановка (с fallback на force-stop).

## Данные и логи

- `users.json` — профили сотрудников, версии, привязки таблиц.
- `states.json` — текущие сценарии диалога (шаги ввода).
- `photo-log.jsonl` — журнал фото пробега.
- `route-sheet-log.jsonl` — журнал маршрутных листов.
- `reconciliation-log.jsonl` — журнал сверок.
- `fun-reactions.json` — сохранённые `file_id` стикеров/GIF для авто-реакций.

## Версии и уведомления об обновлении

- При старте бот проверяет изменения JS-кода и автоматически увеличивает patch-версию (`2.0.x`).
- Если версия изменилась, бот рассылает пользователям уведомление в личный чат.
- Текст уведомления короткий: «что изменили» на 1-3 пункта.
- Можно задать свои пункты через `UPDATE_NOTES` (разделитель `|`).

Пример:

```env
UPDATE_NOTES=🎛 Обновили меню|🔧 Исправили ошибки ввода|⚡ Ускорили ответы бота
```

## Стикеры и GIF на ошибки/успех

## Сверки: наличные к сдаче

- При отправке фото сверки бот пытается считать строку `Наличные X / Y ₽`.
- Если `X > 0` и `Y >= 1,00 ₽`, в подпись фото добавляется строка:
  - `💵 К сдаче в следующую смену: <code>12 897,27 ₽</code>`
- Если `Наличные` пусто, `Y = 0,00 ₽` или сумма меньше `1,00 ₽`, строка в подпись не добавляется.
- Сумма привязывается к сотруднику, который отправил сверку, и показывается напоминанием при следующем старте смены.

- Управление флагом: `FUN_REACTIONS_ENABLED=true/false`.
- Если флаг не задан, бот включает реакции автоматически, когда есть сохранённые стикеры/GIF.
- Бот отправляет реакции на сообщения:
  - ошибки (`❌`, `⚠️`);
  - успех (`✅`).
- Можно включить автоимпорт стикерпаков (по умолчанию включён).
- Достаточно указать ссылки в `FUN_STICKER_SETS` или отправить стикер в личный чат боту: бот запомнит `file_id` и подтянет весь стикерпак.
- Ограничение Telegram: старую историю бот не читает, поэтому для ранее отправленных паков лучше один раз отправить любой стикер из каждого пака после включения.

Пример:

```env
FUN_REACTIONS_ENABLED=true
FUN_REACTION_COOLDOWN_MS=30000
FUN_IMPORT_STICKER_SETS=true
FUN_STICKER_SETS=https://t.me/addstickers/TikTok_cats_animals
FUN_ERROR_STICKERS=CAACAgIAAxkBAAIBQ2example1,CAACAgIAAxkBAAIBQmexample2
FUN_SUCCESS_STICKERS=CAACAgIAAxkBAAIBQ3example3
```

## Частые проблемы

- `Сотрудник не найден` -> проверьте ФИО, как в таблице (без лишних пробелов/ошибок).
- `Таблица не привязана на месяц` -> администратор должен выполнить `/sheet ... active/next`.
- `Нет доступа к таблице` -> выдайте доступ сервисному аккаунту как редактору.
- `Protected cell` в Google Sheets -> снимите защиту нужных ячеек для сервисного аккаунта.
- Фото не уходят в чат -> проверьте `WORK_CHAT_ID` и права бота в группе.

## Архитектура

Файлы проекта:

| Файл | Назначение |
|------|------------|
| `bot.js` | Точка входа, обработчики Telegram, основная логика |
| `googleSheets.js` | Работа с Google Sheets API, lookup курьеров, запись времени/пробега |
| `storage.js` | Локальный JSON-стор: `users.json` (профили) и `states.json` (in-flight состояние) |
| `utils.js` | Чистые функции: округление времени, формат колонок, нормализация ФИО |
| `mileageOcr.js` | Распознавание пробега через RapidOCR + AI vision |
| `sheetCommand.js` | Команда `/sheet` для админов — привязка Google-таблиц |
| `config.js` | Константы: WORKPLACES, DEVICES, лимиты, шаблон таблицы |
| `rapidocr_server.py` | Локальный HTTP-сервер RapidOCR на Python |
| `tests/` | Юнит-тесты (без сторонних зависимостей) |

Хранилище данных:
- `users.json` — профили курьеров, ID Google-таблиц по магазинам/месяцам
- `states.json` — текущее состояние диалога (что юзер сейчас вводит)
- `version.json` — версия бота, обновляется автоматически по хэшу .js файлов
- `backups/` — почасовые копии `users.json` / `states.json` за последние 7 дней
- `*-log.jsonl` — ленты событий: фото, маршрутные листы, сверки, OCR feedback

Обновления и деплой:
- При изменении любого `.js` файла версия автоматически инкрементируется при следующем старте.
- Админам приходит preview обновления с кнопками «Отправить / Изменить / Пропустить».
- Деплой на VPS через GitHub Actions: `git push` в `main` → автоматический pull, `npm install`, `pm2 restart`.

Локальные unit-тесты без сторонних зависимостей. Запуск:
